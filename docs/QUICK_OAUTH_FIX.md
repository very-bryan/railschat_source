# Google OAuth 빠른 설정 가이드

## 🚀 5분 안에 해결하기

### 1단계: Google Cloud Console 열기
[https://console.cloud.google.com](https://console.cloud.google.com)

### 2단계: OAuth 동의 화면
1. 왼쪽 메뉴 → **API 및 서비스** → **OAuth 동의 화면**
2. 현재 상태 확인

### 3단계: 테스트 사용자 추가 (테스트 모드인 경우)
1. 스크롤해서 **"테스트 사용자"** 섹션 찾기
2. **"+ 사용자 추가"** 클릭
3. 본인 Gmail 주소 입력
4. **"저장"** 클릭

### 4단계: 또는 앱 게시 (모든 사용자 허용)
1. 상단의 **"앱 게시"** 버튼 클릭
2. **"확인"** 클릭
3. 도메인 없어도 됨!

### 5단계: 리디렉션 URI 재확인
1. **API 및 서비스** → **사용자 인증 정보**
2. OAuth 2.0 클라이언트 ID 클릭
3. 승인된 리디렉션 URI 확인:
   ```
   http://localhost:3000/users/auth/google_oauth2/callback
   ```
   - 끝에 슬래시(/) 없음
   - 포트 3000 포함
   - http (s 없음)

### 6단계: 브라우저 캐시 삭제
1. 브라우저 시크릿/프라이빗 모드로 테스트
2. 또는 다른 브라우저로 시도

## ❓ 자주 묻는 질문

**Q: 도메인이 꼭 필요한가요?**
A: 아니요! 개발 중에는 localhost로 충분합니다.

**Q: "개발자가 액세스 권한을 부여하지 않았습니다" 오류**
A: 테스트 모드입니다. 테스트 사용자를 추가하거나 앱을 게시하세요.

**Q: Google 인증이 필요한가요?**
A: userinfo.email과 profile만 사용하면 인증 불필요합니다.

**Q: 프로덕션 배포 시?**
A: 그때 도메인 추가하고 https 리디렉션 URI 추가하면 됩니다.
